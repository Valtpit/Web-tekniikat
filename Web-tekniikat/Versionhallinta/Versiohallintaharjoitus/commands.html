Some version control commands

  • git init
    Create a new local repository.
  • git status
    List the files you have changed and those you still need to add or commit.
  • git remote add origin <server>
    If you have not connected your local repository to a remote server, add the server to be able to push to it
  • git remote -v
    List all currently configured remote repositories.
  • git push origin master
    Send changes to the master branch of your remote repository
  • git add <filename> or git add .
    Add one or more files to staging (index).
  • git commit -m "Commit message"
    Commit changes to local repository but not yet to the remote repository.
  • git commit -a
    Commit any files you have added with git add, and also commit any files you have changed since then.
  • git pull origin master
    Fetch and merge changes on the remote server to your working directory master branch.
  • git log
    Get a listing of commits.
  • git branch
    Get a listing of branches.
  • git branch name
    Create a branch.
  • git checkout name/master
    Switch to specific branch or master branch.
  • git push -u origin name
    Push a new branch to a remote repository.
  • git merge master name
    Merge branch to a master branch.
